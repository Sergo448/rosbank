При выполнении данной работы была выбрана среда разработки DBeaver.

Предварительно на рабочий ноутбук был установлен необходимый софт для работы
с СУБД PostgreSQL.
Через cmd была создана база данных 'rosbank' и пользователь 'sergey' которому
были предоставлены все права с помощью инструкуции 'GRANT...'

В DBeaver было создано подключение к локальному хосту на 5432 порте. Через который
был создан внутренний sql script. В нем была создана схема training.
Пример создания схемы вы можете пронаблюдать в файле create_schema_traiding.sql

Кроме того, были созданы еще дополнительные представления и функции, которые можно также увидеть среди скриптов.

Далее была проведена проверка на добавление значений в случайную таблицу на
примере скрипта insert_check.sql

            ПОЯСНЕНИЕ ТАБЛИЦ
a. trading.securities, которая отражает баланс клиента на текущую дату и содержит следующие поля
    • client_date - Дата
    • client_id - Id клиента
    • client_ISIN_paper - ISIN бумаги
    • client_n_papers - Кол-во бумаг в портфеле
    • client_currency - Валюта
    • client_rialto -Биржа
    • client_closing_price - Цена Закрытия
б. traiding.clients - которая содержит информацию о инвесторах.
    • investor_id - ID инвестора / клиента
    • full_name - ФИО инвестора / клиента
    • investor_age - возраст клиента
    • investor_country - страна клиента
    • number_in_briefcas - количество бумаг в портфеле
    • positive_capital - положительный капитал
    • qual_or_nonqual - квалифицированный или неквалифицированный инвестор
    • personal_manager_id - ID персонального менеджера.
с. traiding.investment_currencyhistory - которая содержит информацию о валютных котировках.
    • currency - валюта
    • direct_quote -  прямой курс
    • reverse_quote - обратный курс
    • cross_quote - кросс
    • quote_date - дата получений курса
д. traiding.history_prices - которая содержит информацию о котировках ценных бумаг на разных биржах в разрезе дней
    • currency - валюта
    • direct_quote -  прямой курс
    • reverse_quote - обратный курс
    • cross_quote - кросс
    • date_ - дата котировки
    • time_ - время котировки

Все столбцы были заданы в соответствии со своим название соответствующим типом данных.
